使用github-page搭建静态网页

1.创建新repositories

注意！！！ 一定不要勾Add README file选项

 

否则会出现报错

error: src refspec master does not match any. error: failed to push some refs to 'git@github.com:hahaha/ftpmanage.git'

2.进入仓库，选择Clone with HTTPS，然后复制http地址

3.进入本地操作

  • 进入本地文件夹，空白处右击选择GIT Bash Here
  • 把github上所创建的仓库克隆到本地

git clone https://github.com/272TNT/depression2.git（https://github.com/272TNT/depression2.git替换成此时所需的对应的网址）

  • 上述步骤以后你的本地项目文件夹下面就会多出个文件夹，该文件夹名即为你github上面的项目名，如图我多出了个depression2文件夹，我们把本地项目文件夹下的所有文件（除了新多出的那个文件夹不用），其余都复制到新多出的文件夹下，

 

  • 接着输入命令cd depression,进入depression2文件夹

  • 接下来依次输入以下代码即可完成其他剩余操作：

    git add . （注：别忘记后面的 . ，此操作是把Test文件夹下面的文件都添加进来）

    git commit -m "提交信息" （注：“提交信息”里面换成你需要，如“first commit”）

    git push -u origin master （注：此操作目的是把本地仓库push到github上面，此步骤需要你输入帐号和密码）

4.进入github网页中对应仓库

  • 选择settings
  • 选择Page，选择启用github-page

 

Git ssh密钥创建、重置和配置

1.查看是否配置过密钥

在git bash窗口输入cd ~/.ssh

输入ls查看秘钥列表

如上图是已经配置过了就无需配置。如果没有看到id_ras.pub文件即没有配置，然后请看下面如何配置。

2. git创建密钥

$ cd ~

$ ssh-keygen.exe

三次回车

3.打开对应目录的 id_rsa.pub 文件

注意：id_rsa.pub文件在电脑C盘，一般路径为C:\Users\weibu.ssh

4.打开id_rsa.pub文件

复制文件中的内容，粘贴到GitHub账号下的SSH keys,按图示操作即可
